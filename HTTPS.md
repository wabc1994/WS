# http缺点
1. 通信使用明文(不加密)，内容可能会被窃听 (窃听风险)
2. 不验证通信方的身份，因此有可能遭遇伪装 （篡改风险）
3. 无法证明报文的完整性,所以有可能已遭篡改 （冒充风险）

换一种说法

1. 第三方可以获取通信内容
2. 第三方可以修改通信内容
3. 第三方可以冒充他人身份参与通信


# https解决方案
1. 所有信息都是通过加密传输，第三方无法进行窃听
2. 具有校验机制，一旦被篡改，通信双方都可以发现
3. 具备身份证书，防止身份被冒充

# 如何实现
增加一个层，在http应用层和tcp 传输层之间增加一层加密和解密协议


# https的缺点
- 加密解密过程复杂，导致访问速度慢
- 加密需要认向证机构付费
- 整个页面的请求都要使用HTTPS**(一般一个页面有多个http请求)**
